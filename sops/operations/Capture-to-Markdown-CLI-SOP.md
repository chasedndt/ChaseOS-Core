## Allowed Inputs

- `--text TEXT` for operator-supplied visible text.
- `--from-file PATH` for an explicit local text or Markdown file.
- `--from-html PATH` for an explicit local saved HTML file.
- `--from-controlled-html PATH` for a Browser Runtime or Studio webview HTML
  artifact under an allowed vault-local artifact directory:
  `docs/framework-logs/Browser-Runs/`, `runtime/browser_runtime/artifacts/`, or
  `runtime/studio/webview_artifacts/`.
- `--from-screenshot PATH` for an explicit local `.png`, `.jpg`, `.jpeg`, or
  `.webp` screenshot attachment under an allowed vault-local evidence
  directory: `docs/framework-logs/Operator-Screenshots/`, `docs/framework-logs/Browser-Runs/`,
  `runtime/browser_runtime/artifacts/`, or
  `runtime/studio/webview_artifacts/`.
- `--from-stdin` for piped or redirected text.

This CLI does not capture active windows, active browser tabs, clipboard state,
screen pixels, live screenshots, optical character recognition output, browser
history, cookies, sessions, or credentials. Pass 8 records those and related
external surfaces as explicitly deferred and not authorized.

Controlled HTML requires `--url` with an `http` or `https` URL. Use
`--allowed-origin` when a workflow has declared an origin boundary. The CLI
blocks controlled artifacts that contain forms, password/file inputs, download
links, browser-history markers, cookie access, or local/session storage access.

Screenshot attachment import validates only the selected local file path,
allowed evidence directory, image extension/signature, size, SHA-256, and a
byte-level nonblank posture. It records the attachment as
`operator-review-required`, marks optical character recognition as
`not_performed`, and blocks cloud optical character recognition.
It cannot determine whether private/authenticated content appears inside the
image. On save, copied screenshot attachments are marked
`retain_until_operator_review` and `pending-review`; cleanup or deletion
requires a separate operator decision and is not performed by this CLI. Pass 11
adds `attachment_disposition_policy` with metadata states `retain`,
`retain-until-downstream-review`, `needs-redaction`, and `delete-requested`.
`delete-requested` is metadata only and does not enable runtime deletion,
Studio delete controls, or cleanup execution.

## Write Boundary

- `preview` is no-write.
- `recent` is read-only over Phase 8 quarantine sidecars.
- `save` writes only raw intake artifacts under `03_INPUTS/00_QUARANTINE/`:
  Markdown, Phase 8 `.meta.json`, optional `.visual_capture.json`, and dedup
  registry state.
- `review` writes only review-state metadata to the Phase 8 `.meta.json`
  sidecar and adjacent `.visual_capture.json` packet when present. It does not
  rewrite captured Markdown content.
- Studio Capture Markdown review controls call the same review writer through
  `review_capture_to_markdown`. The Studio clickthrough is sidecar/packet-only
  and does not grant content rewrite, source-package, AOR, SIC, graph, provider,
  external-send, attachment cleanup, or canonical-promotion authority.
- `acquisition visual-capture-downstream-gate` is read-only. It evaluates one
  reviewed VCMI artifact against the acquisition preview bridge, source-pack
  builder workflow manifest, AOR task type table, and SIC source-package schema.
  It does not write source packs, approval artifacts, Agent Bus tasks, AOR
  events, SIC records, graph indexes, or canonical knowledge.
- `acquisition visual-capture-source-pack-approval-preview` is read-only. It
  reuses the downstream gate, returns a deterministic request digest, required
  operator statement, read scope, and future write target summary, and writes no
  approval artifacts, source packs, Agent Bus tasks, AOR events, SIC records,
  graph indexes, or canonical knowledge.
- `acquisition visual-capture-source-pack-write` is write-capable only after the
  request digest matches the approval preview and the operator statement exactly
  matches the required statement. It writes create-only source-pack JSON
  artifacts plus an exact-once marker under `runtime/acquisition/packs/`. It
  does not write approval artifacts, Agent Bus tasks, AOR events, SIC records,
  graph indexes, canonical knowledge, provider calls, external sends,
  attachment cleanup/deletion, or the original VCMI sidecar/content/packet.
- `acquisition visual-capture-source-pack-aor-dispatch-readiness` is read-only.
  It runs only after source-pack write, verifies the exact-once marker,
  persisted source-pack artifacts, source-pack-builder workflow, AOR task type,
  source-pack-builder role card, and AOR engine handler registration, and
  returns a future dispatch packet preview. It does not write Agent Bus tasks,
  call AOR, run AOR dry-run, write AOR/OSRIL audit records, ingest SIC, mutate
  graph/canonical state, call providers, send externally, or delete
  attachments.
- `acquisition visual-capture-source-pack-aor-dispatch-approval-design` is
  read-only. It runs only after AOR dispatch readiness succeeds and returns a
  deterministic future approval packet preview for reviewed source-pack AOR
  dispatch. It does not write approval requests, approval artifacts, approval
  decisions, approval consumption records, approval exact-once markers, Agent
  Bus tasks, AOR/OSRIL audit records, SIC records, graph indexes, canonical
  knowledge, provider calls, external sends, or attachment cleanup/deletion.

and deletes no attachment.

`acquisition visual-capture-source-pack-canonical-promotion-approval-consumption`
reads one completed graph-indexing execution artifact, the pending
canonical-promotion approval request, and the written approval decision
artifact. It can consume one approved or rejected decision only after exact
graph-indexing artifact, optional graph snapshot, optional canonical-promotion
candidate, approval-request, approval-decision, approval-consumption, decision
value, and operator-statement checks match. It writes the exact-once consumption
marker before the create-only consumption artifact. Approved consumption marks
the path ready for a future canonical-promotion executor; rejected consumption
keeps the path closed. It writes no canonical knowledge, mutates no graph state,
calls no provider, sends nothing externally, and deletes no attachment.

## Secret Handling

Secret-like source text blocks save by default. Use
`--allow-secret-redaction` only when the operator explicitly wants a redacted
raw quarantine artifact. The raw secret value must not be written to vault
content, logs, sidecars, or generated docs.

Secret-like review notes or reviewer labels block `review` by default. Use
`--allow-secret-redaction` only when the operator explicitly wants redacted
review metadata; the unredacted secret value must not be written.

## Deferred External Surfaces

Pass 8 exposes policy id `vcmi.external_capture_surfaces.deferred.v1` through
the Studio Capture Markdown panel and shell registry. It is a readiness and
governance contract only.

The following surfaces are `deferred`, `not_built`, and
`authority_allowed=false` for the command-line interface and ambient capture
paths:

- Operating-system global keyboard shortcut capture.
- Overlay capture user interface.
- Ambient clipboard capture and background clipboard monitoring.
- Active-window capture.
- Ambient screen-pixel recording or background screenshot capture.
- Discord command capture.
- External control-plane capture.
- External active-browser-tab capture.
- Accessibility-tree capture.

Do not operate Capture to Markdown as an operating-system keyboard shortcut
listener, overlay, Discord bot, external command endpoint, active-window reader,
screen recorder, browser-tab reader, accessibility-tree reader, or ambient
clipboard monitor. The Studio-only explicit screen collector and explicit
clipboard text collector are separate Settings-gated and click-only product
surfaces. The explicit browser artifact collector is also separate and accepts
only an operator-selected ChaseOS-owned artifact with a declared source address;
it does not read live browser tabs, browser profiles, browser sessions, cookies,
or browser history. The explicit ChaseOS-owned browser page collector is separate
too; it launches only an isolated ChaseOS-owned browser runtime for a declared
address after Settings enablement plus a Capture action, and it does not read a
personal browser tab, profile, session, cookies, storage, or history. The
explicit Discord artifact collector is separate too; it accepts only an
operator-selected ChaseOS-owned Discord records artifact with a declared Discord
source and does not read tokens, webhooks, bindings, live events, or call
Discord. These collectors do not change this command-line boundary.
Future work on any ambient surface requires separate design, operator approval,
privacy review, and tests.
