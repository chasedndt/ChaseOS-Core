# Core vs Personal

ChaseOS separates reusable framework material from private operating truth.

## Core

Core is public/forkable framework material:

- architecture docs;
- governance model;
- runtime/agent standards;
- workflow packaging standards;
- neutral templates;
- synthetic examples;
- safe setup and CLI docs.

## Personal

Personal is the private live instance:

- active projects;
- source notes and research;
- logs and operator briefs;
- credentials/config;
- agent-bus state;
- runtime memory;
- personal doctrine and decisions.

## Export rule

Nothing moves from Personal to Core merely because it exists. A Core candidate must be reusable, sanitized, navigable, scanner-clean, and review-approved.
