# Bootstrap Checklist

Use this checklist when creating a new ChaseOS Personal instance from Core.

## Before copying templates

- [ ] Decide public Core repo path.
- [ ] Decide private Personal workspace path.
- [ ] Confirm they are not the same folder.
- [ ] Confirm credentials will live outside tracked markdown.
- [ ] Confirm raw inputs and logs will not be committed to Core.

## Create private workspace

- [ ] Create home/index note.
- [ ] Create project area.
- [ ] Create knowledge/source/synthesis area.
- [ ] Create logs/audit area.
- [ ] Create runtime config/state area outside public templates.

## First governance pass

- [ ] Read Permission Matrix and Trust Tiers.
- [ ] Define approval roles.
- [ ] Define writeback destinations.
- [ ] Add a first decision log.
- [ ] Run a no-external-side-effects test workflow.
