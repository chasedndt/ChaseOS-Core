# Workflow Exchange Readiness Standard

A workflow is exchange-ready when another operator can understand, inspect, and safely adapt it.

## Readiness gates

- neutral description;
- declared inputs/outputs;
- no secrets or private paths;
- synthetic examples;
- verification proof;
- license/usage terms;
- failure and rollback notes;
- approval requirements.

Exchange-ready does not mean live-approved. Live execution remains instance-specific.
