# Domain Playbook Standard

A domain playbook adapts generic ChaseOS workflow patterns to a specific domain without leaking private data.

## Contents

- domain scope;
- trusted/untrusted inputs;
- common workflows;
- source quality rules;
- output templates;
- review criteria;
- escalation paths.

## Public-safe examples

Use synthetic projects, placeholder sources, and neutral claims. Do not embed real client, financial, credential, or personal operating data.
