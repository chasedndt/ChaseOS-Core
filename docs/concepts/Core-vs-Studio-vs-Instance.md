# Core vs Studio vs Private Instance

## ChaseOS Core

The reusable public framework: docs, templates, schemas, runtime contracts, governance rules, and source-safe starter code.

## ChaseOS Studio

The application layer built above Core. Studio renders Core contracts as panels, workflows, approvals, runtime status, and evidence views.

## Private Instance

The operator's real notes, projects, runtime queues, credentials, logs, schedules, and local configuration.

## V1 Packaging Rule

Public Core can ship with Studio contracts and reviewed source-safe app code. The packaged `.exe` installer should ship as a release artifact with checksums and release notes, not as a normal committed binary.
