# Principles Example

## Operating Principles

- Keep source material separate from synthesis.
- Keep generated material separate from canonical material.
- Require approval for durable writes.
- Preserve provenance for promoted knowledge.
- Prefer bounded runtime tasks with clear evidence.

## Decision Rules

- If authority is unclear, pause and request review.
- If an output changes system truth, write an audit note.
- If a runtime touches external systems, record adapter scope first.
