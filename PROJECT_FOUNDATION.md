# ChaseOS Project Foundation

ChaseOS is a framework-first control plane for human-AI work. It treats memory, runtime behavior, source intake, and writeback as governed system surfaces rather than loose files.

## Foundation Principles

- The repository is the source of truth.
- Runtimes are adapters, not owners.
- Writes must be explicit, bounded, reviewable, and auditable.
- Generated material is not canonical until promoted.
- Runtime authority is declared before execution.
- Public Core remains separate from private deployment state.

## Core Subsystems

- Memory and knowledge organization.
- Source intelligence and capture workflows.
- Agent registration and permission boundaries.
- Gate and approval patterns.
- Operator run evidence.
- Export-safe templates for public distribution.

## Implementation Boundary

Core describes patterns and starter materials. A private deployment may add vault content, schedules, workflow runs, runtime logs, model adapters, and local automation while preserving the same governance rules.
