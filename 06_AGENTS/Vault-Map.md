# Vault Map

This map describes a public Core layout. Private deployments may extend it while keeping local content and runtime state separate.

## Public Folders

- docs: public examples and framework notes.
- templates: reusable note and audit templates.
- runtime: public schemas and adapter examples when included.
- core_export: export manifest, reports, and curated templates.

## Private Deployment Folders

Private vaults can add home, project, knowledge, input, log, archive, and runtime-state folders. Those folders should not be exported to Core unless sanitized or templated.
