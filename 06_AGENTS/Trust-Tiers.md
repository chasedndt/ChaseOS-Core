# Trust Tiers

Trust tiers help separate raw material, generated drafts, reviewed suggestions, promoted notes, and canonical system truth.

## Example Tiers

- raw: captured but not reviewed.
- suggested: generated or inferred and awaiting review.
- promoted: accepted into a working layer.
- canonical: accepted as system truth.
- archived: no longer active.
- disputed: kept for traceability while under review.

Trust changes should be explicit and auditable.
