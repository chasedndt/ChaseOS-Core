# Adapter Manifest Standard

An adapter manifest describes what a runtime can do and what it must not do.

## Required Sections

- Identity.
- Capabilities.
- Allowed inputs.
- Allowed outputs.
- Forbidden surfaces.
- Review and approval requirements.
- Test or readiness checks.

## Publication Boundary

Public manifests should contain no private endpoints, credential values, live identifiers, or machine-local paths.
