# security

`security` holds security-audit indexes, public security policy material, and review templates.

Public Core should include reusable security guidance and example indexes. Live security audit findings from a private deployment may contain local paths, host details, system configuration, or runtime weaknesses and must be reviewed before publication.

## Public Core Contents

- `Security-Audits-Index.example.md`

## Private Deployment Rule

Publish policy and templates first. Publish live audit findings only after a separate redaction review.
